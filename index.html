Intro

Introduction

I'm a sofware developer with almost 18 years of experience. Working in a business like software development/engineering exposes you to huge amount of emerging technologies. Here are some of the technologies that I'm currently working with and most excited about.

Micro-Services

I like my code to focus on doing one thing really, really well. Micro-Services allow me, as a developer, to create small units of well tested functionality that together comprise a vast ecosystem of business resources. I'm a long time Spring Boot user. I believe it provides great built in capabilities and a starting point in any project to develop consistent and reliable services.

Containers

Containers represent a fantasitc way for developers to deliver code in consistent and portable manner. I can build and deploy container based services to multiple operating systems, hardware configurations and environments with a high degree of certainty that they will behaive in a similar way everywhere.

Orchestration

I'm a fan of small services that are subject matter experts in the function they perform, which can lead to a large number of services. However, I'm also able to package my services reliably as container images, and distribute my services across vast cloud resources using tools like Kubernetes. I've spent 3 years running production clusters with thousands of workloads.

Projects

Some of my more recent projects and technologies involed during development.

Runner

Runner is a cloud based provisioning and configuration tool for the CenturyLink Cloud platform. Runner, based on Ansible, allows users to define infrastructure and applications in an idempotent manner. Through the use of Ansible as well as a custom created Ansible connector, Runner is able to provision and manage cloud assets in the user's account space without the use of an agent. As the Senior Lead Developer for the project, I led the development of the proof of conecpt and production versions of the project. Read more about Runner here.

Multi-DC Kubernetes

As a major advocate for containerization and Kubernetes, I was asked to create a Multi-Datacenter Kubernetes intallation that could be utilized by all teams across our CenturyLink Cloud product organization. This platform is currently in use in multiple installations, including a global cluster spanning 17 data centers. This custom cluster allows teams to deploy services in a consistent way to any datacenter in CenturyLink Cloud, simply by using a label. Additional information on the technologies that are part of the cluster is available here.

Portr

Running a large number of containers requires access to a highly available container registry. To eliminate the need for each team in the CenturyLink Cloud organization to create and maintain an independent container registry, I constructed Portr. Portr is a secured, multi-tenant container registry providing private repositories and image access controls. In addition, I created Portr to provide in-depth security analysis of all images and act as a quality gate for deployments. Additional info on Portr is available here.

Padloq

Padloq is a personal project that was created as an Identity As A Service solution for the larger Podleo platform. The goal of Padloq is to simplify OAuth and was constructed to provided authentication and identity resolution via a number of configurable downstream providers like Google, Facebook, LinkedIn, GitHub and Twitter. Padloq was a ton of fun to develop and continues to grow. You can read more about Padloq here.

Podleo

Running containers at scale is a large undertaking, with vast benefits. It's complex and involves multiple technologies. Podleo is a personal attempt to bring the power of this technology to the non-technical consumer. Podleo is being developed as a multi-tenant, multi-site platform to simplify this technology through API access as well as a simplified user interface. You can read more about Podleo here.

Recordr

Recordr is a project that is still in the proof of concept phase. It will provide simple key/value as a service operations for our Podleo platform. This cloud-native service will allow users to set and query simple data fields, as well as provide accumulation fields. I intend to leverage this service to process the Podleo usage data stream. I like to think of all potential solutions as products for the platform.
